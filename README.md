## How I work

My debugging and delivery process is deliberately engineering-led:

1. Reproduce the failure or define the missing reproduction path.
2. Isolate the root cause instead of patching symptoms.
3. Identify the smallest safe fix.
4. Add a test, verification command, or regression note where practical.
5. Document what changed, why it changed, and how to prevent recurrence.
6. Leave the repo in a more maintainable state than I found it.

## Operating principles

- Architecture before activity.
- Evidence before opinion.

- State machines over hidden assumptions.
- Runbooks over heroics.
- Observability over guessing.
